## Workflow

1. Draft new content in `launch/content/`
2. Review and finalize
3. Move final versions to `publications/` (for documents) or website repo (for web content)
4. Update status tracker in `LAUNCH_PLAN.md`

## Session Protocol

When working on launch tasks:
1. Read `LAUNCH_PLAN.md` Section 10 (Status Tracker) first
2. Pick the highest-priority NOT STARTED or IN PROGRESS item
3. Update status to IN PROGRESS before starting
4. Update status to COMPLETE when done, with session number
5. Log any decisions in Section 11 (Decision Log)
6. Add session to Section 12 (Session History)
